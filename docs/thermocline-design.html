Conductor design · v4 · cold-score → cold-epoch → thermocline

Thermocline

Attention-gated, LLM-quality compression that moves in deliberate epochs, not a constant trickle. It starts preparing the next state early, decides the whole thing slowly and off to the side, then swaps it in all at once — while a hard budget floor guarantees the agent never overflows.
updates discrete atomic epochs invariant live ≤ budget, always digests LLM (host.complete), recoverable signal Qwen-0.5B attention deep history bounded · re-compressible type external · WebSocket parents attention-folder × compaction-naive
what changed since v2 — two flaws you caught

① The fold digest was mechanical and lossy (the engine's per-kind stub, zero LLM) — and it lived in the most-populated layer. ② Immutable strata break the budget: they only accumulate, so over a long session their sum alone eventually exceeds the ceiling with no way to reclaim it. The design is rebuilt around fixing both — and they turn out to share one keystone.

the keystone — prepare ahead, commit all at once

Thermocline runs like double-buffered graphics: it computes the entire next state off to the side — every fold decision and every LLM summary — while the agent keeps seeing the current, byte-stable state. When the whole next state is ready, it swaps in atomically, in one update, and drops back under budget. Between swaps, nothing changes — no continuous compaction, no per-turn fiddling. The budget guarantee survives because each epoch is started early, with one instant deterministic fallback for a sudden burst.

01The budget invariant — the spine of the whole design

A conductor that can drift over budget is broken, full stop. Thermocline's first commitment, above relevance, above cache, above everything: the agent is never over budget. It keeps that promise with deliberately-timed epochs — each one started early and prepared off to the side — backed by a move that always frees space, terminating in a hard delete.

Why it provably terminates — rung 4 strictly reduces tokens and is available whenever any stratum exists, so the loop cannot get stuck above budget. It bottoms out at "protected tail + one minimal stratum." Below that is the human's budget/tail dials, not the conductor's to override.

Attention decides order; budget decides depth

The same split the garbage-collector conductor uses (signal picks priority, budget picks how far to go). Attention spares the still-used content longest; the budget invariant decides how deep the next epoch must reach. Relevance never overrides the ceiling — it only orders who gets compressed first.

Anticipatory epochs, one instant safety epoch

A planned epoch starts early (around eighty percent full), so its slow LLM work finishes before the ceiling. If a sudden burst would breach budget before the planned epoch is ready, a deterministic emergency epoch applies instantly — one atomic, mechanical compression, no LLM, no waiting — and the high-quality epoch still swaps in afterward. That instant fallback is the only reactive move, and it is what keeps "never over budget" a guarantee rather than a hope.

HOLD
Steady, byte-stable applied state, re-sent unchanged each turn. Cache warm. No actions.
→
PREPARE · early
Cross ~80%: compute the whole next target off to the side and fire every LLM summary in parallel. Visible state unchanged.
→
READY
All summaries returned. The complete next state sits assembled in the back buffer.
→
COMMIT · atomic
Swap in all at once — one update, drop to low-water, one deliberate cache miss. → back to HOLD.
Double-buffered. The agent only ever sees a finished, stable state; the slow LLM work happens during PREPARE while the old state holds — so an epoch is felt as a single clean transition, never a trickle. EMERGENCY (rare): if a burst outruns PREPARE, a deterministic epoch commits instantly to hold the line, and the planned epoch still lands after.

02The two parents

attention-folder

A 0.5B probe scores structural gravity — how much the working tail attends back to each block — and folds the least-attended.

▲ a learned signal of what the model actually needs; cache-warm epochs; reversible folds.

▼ only ever folds to a shallow per-block digest, only ever grows the fold set, never re-surfaces a block, and eventually runs out of reversible room.

compaction-naive

At ~90% it summarizes the aged region into one LLM prose summary and collapses it into a group. User messages verbatim.

▲ real token reduction; human intent preserved verbatim.

▼ blind (compacts by age), irreversible (no unfold), and recursively amnesiac (summarizes its own summary, compounding loss).

the synthesis, in one line

Give the deep-compression engine an attention admission test and LLM-quality intermediate digests; stage every irreversible step behind a reversible, recoverable probation; and bound the result so it can never outgrow the budget.

The host passes a conductor's custom fold digest verbatim, and resolveUnfold/resolveRecall match the agent's code against foldCode(id) — independent of the body text. So Thermocline prefixes each LLM digest with {#${'{'}foldCode(id){'}'} FOLDED} (obtained from host.digestOf(id)) and the agent can unfold/recall it exactly as a normal fold. And recall on a group returns members' original .text regardless of the group's summary — so strata are recall-able by construction.

04Bounded deep history — why nothing is immutable

Fix #2. The deep-history zone (strata) lives under a fixed token ceiling. New strata push in at the warm end; when the zone would overflow, the oldest, coldest strata merge into coarser ones, and drop off the cold end as the floor. The zone's size is therefore constant no matter how long the session runs.

A multi-resolution memory. Recent strata are fine-grained; ancient ones are merged coarse. Total tokens are capped at D; the number of resolution levels grows only ~log(session length). Attention spares the still-used; the rest blurs with age.

Forgetting is necessary — the design is its shape

Under a fixed budget and unbounded history, some information must be shed — that's information theory, not a flaw to engineer away. Naive compaction sheds it as uniform recursive blur over everything aged. Thermocline sheds it as recent-sharp, ancient-blurred, attention-spared — and bounded-log recursion confined to the coldest content, not linear recursion over all of it.

vs v2's immutable strata

v2 froze each stratum forever. Lovely for cache and amnesia — fatal for budget: N strata over a long session sum past the ceiling with no reclaim. v3 trades a little cache stability (rare deep-merge flushes) and a little ancient fidelity (graded blur) for the thing that actually matters: the agent is never over budget.

05Attention & the double gate

Attention sets the order at every rung — coldest compressed first, warmest spared longest. And it guards the one irreversible-to-the-standing-view step (L2 → L3) with two independent cold votes, one of them behavioral.

Attended-to, or in the protected tail.
→
Trim → Digest
Attention dropped. Reversible, recoverable, LLM-quality. Dwell clock starts.

Stratum
Deep, recall-able, re-compressible.
BOTH, sustained ≥ K epochs:
  • ① probe temperature cold (re-scored fresh this epoch)
  • ② agent did not recall/unfold it while it sat folded
↩︎ Any re-warm resets to Full: the probe re-attends · the agent unfolds/recalls · ever-warm units need 2K, not K. Float-up is epoch-granular: each epoch re-plans folds from current temperatures, so a unit whose attention recovered is simply not re-folded — it returns to full at the next commit. No continuous re-surfacing (that would violate the epoch model).
Why a behavioral second gate — gate ① is a 0.5B proxy with known recency bias; betting an irreversible step on it alone is unsafe. Gate ② is the agent's own conduct: it had the digest and the recovery tag in front of it and chose not to pull the content back.

Per-run sedimentation (tool-pairs atomic, whole-message snap, hot/pinned units as buoyant run boundaries) is unchanged from v2 — it keeps every group command wire-valid and dissolves the "one hot block traps a cold region" failure by compacting each cold pocket independently.

09Design evolution

v3 · churn
Budget was enforced every pass, but LLM summaries landed one at a time — each a separate visible change. That's continuous action: a trickle of little cache misses, exactly the texture to avoid.
→
v4 · epochs
Double-buffered. Prepare the entire next state (every summary) off to the side while holding steady, then commit atomically in one update. Anticipatory timing + one instant deterministic emergency epoch keep the budget guarantee hard.
v2 · broke
Probation layer folded to the engine's mechanical per-kind stub. Highly lossy, zero LLM, in the most-populated layer.
→
v3 · fixed
LLM digests at L2 (content-hash cached, async upgrade-in-place), Trim as the deterministic placeholder/fallback. Verified recoverable with the existing tag mechanic.
v2 · broke
Immutable append-only strata. Accumulate without bound → eventually exceed budget with no reclaim. A category error.
→
v3 · fixed
Bounded, re-compressible deep zone (merge oldest → drop-oldest floor) + the synchronous-deterministic / async-LLM split. Hard budget invariant.
v1 · broke
Single running front-summary rewritten each pass (full cache flush); blind, irreversible compaction; collaborative posture corrupted bookkeeping; one stale score gated deletion.
→
v1→v2
Per-run sedimentation, double-gate graduation, lock human-steering / keep agent-unfold, no-probe ⇒ no compaction, persisted state. (carried into v3)

10Honest residual risks

inherent
Ancient detail is genuinely lost
Bounded memory means graded forgetting is real, not reversible. recall only works while a stratum exists; once merged/dropped, that detail is gone. This is the information-theoretic price of a hard budget — the design chooses the shape of the loss, it cannot abolish it.
serious
Attention-biased forgetting
The probe's recency/cross-window bias skews which cold content blurs first toward old, far-from-tail blocks. The double gate, ever-warm 2K, and recall reduce — not eliminate — this.
moderate
LLM digest spend & latency
One cached call per folded unit plus per-stratum summaries. Off-path and budget-independent, but real cost and a quality lag (the deterministic form shows until the LLM lands).
moderate
Each epoch commit is a cache miss
A commit changes the context, so the prompt cache rebuilds from that point. Epochs are deliberately infrequent (one clean miss each) — far better than a per-turn trickle — but a burst that triggers the emergency epoch spends an extra one.
minor
Guarantee bottoms at the tail floor
If the human sets budget below "protected tail + one minimal stratum," no conductor can comply — that's the budget/tail dials (human-owned, never lockable), not Thermocline.

11Spec appendix

conduct() — epoch-based, double-buffered

// double buffer: what the agent sees now, and the next state being prepared off to the side
state = { applied, prepared: null, preparing: false }

fun onUpdate(view):                       // called each turn — usually just re-sends
  cap  = min(view.budget, view.contextWindow)
  fill = project(view, state.applied)        // tokens under the CURRENT applied state

  // 1 · ANTICIPATE — begin preparing the next epoch EARLY, off to the side
  if fill >= warmWater·cap && !state.preparing && needNewEpoch():
     state.preparing = true
     target = planEpoch(view)                // run the §01 ladder to a target set (down to low-water)
     fireEverySummary(target)                // all LLM digests + stratum summaries, in parallel
        .allSettled().then(() => { state.prepared = assemble(target); requestRerun() })

  // 2 · COMMIT — swap the WHOLE next state in atomically, once it is fully ready
  if state.prepared:
     state.applied = state.prepared; state.prepared = null; state.preparing = false
     return emit(state.applied)            // ONE atomic swap → under budget, one cache miss

  // 3 · SAFETY — instant deterministic epoch if a burst would breach before prepared is ready
  if fill > cap:
     state.applied = planEpoch(view).deterministic()   // mechanical, no LLM, atomic; LLM epoch still lands later
     return emit(state.applied)

  // 4 · HOLD — re-send the identical applied state (byte-stable, cache warm)
  return emit(state.applied)

project() = liveTokens − Σ folded(tokens−foldedTokens) − Σ strata(tokens−summaryShare), from explicit foldedIds/strata sets (no view-flag inference). tag(id) = {#${'{'}foldCode(id){'}'} FOLDED}, so every LLM digest stays unfold/recall-able. assemble() reconciles against reality at commit time — any unit the agent recalled, unfolded, or re-warmed during PREPARE is dropped from the epoch before it is swapped in.
