01The two parents — one strength, one fatal flaw each

Both conductors already exist in the repo. Each is excellent at one half of the problem and structurally bad at the other half. Thermocline is built on the observation that each parent's strength is precisely the cure for the other's flaw.

attention-folder · external, GPU probe

A Qwen2.5-0.5B probe measures structural gravity — how much the working tail's last-token readout attends back to each earlier block. Folds the least-attended blocks at hysteresis epochs.

▲ strength
  • A learned signal of what the model actually needs — semantic dependency, not lexical overlap or age.
  • Epoch / hold model keeps the prompt prefix cache-warm between fold events.
  • Folds are reversible ({#code FOLDED} tags); the agent can unfold.
▼ fatal flaw
  • Shallow + one-directional. It only ever folds to a per-block digest — it never achieves deep "whole region → one paragraph" compression, and it only grows the fold set. A long session eventually runs out of reversible savings, and a block that becomes relevant again never re-surfaces on its own.

compaction-naive · in-process, host.complete

At ~90% of budget it summarizes the whole aged region into one LLM prose summary and collapses it into a single group(digest). User messages kept verbatim.

▲ strength
  • Real token reduction. A 40-block region becomes one paragraph — compression no per-block fold can match.
  • Human intent survives: user messages reproduced verbatim, Claude-Code /compact style.
▼ fatal flaw
  • Blind + irreversible + amnesiac. It compacts by age, with no notion of importance — it will summarize away a block the model is actively depending on. Once compacted there is no unfold/recall, and each pass re-summarizes a summary (recursive amnesia) so errors compound.
the symmetry

Attention has the signal but only does shallow, reversible compression. Naive compaction does deep, real compression but is blind and irreversible. Give the deep-compression engine the attention signal as an admission test, and stage it behind a reversible probation period — and both flaws cancel.

02The insight: attention is admission control for irreversibility

The load-bearing idea — the thing neither parent can express — is to use attention not to decide folding, but to decide who is allowed to be deleted.

Reversible is cheap

Folding to a tagged digest costs little and is fully recoverable. So fold generously and early — this is just a staging area, not a commitment.

Irreversible must be earned

Summarizing-away is permanent. A block earns it only by proving it is cold — over time, by two independent signals — while sitting safely in the reversible stage.

Two signals, one behavioral

The 0.5B probe is a proxy. So we don't trust it alone: graduation also requires that the agent itself declined to recall/unfold the block while it had the chance.

thesis

Attention's reversible-fold signal becomes the admission test for naive compaction's irreversible deletion. attention-folder never deletes; compaction-naive never measures. The pipeline between them — probation, dwell, double-gate, sedimentation — is the synthesis, not a bolt-on.

03Mechanism

3.1 · Temperature — the signal

temperature(unit) ∈ [0,1] is the attention mass the probe assigns a block against the current working tail ("where the agent is heading"). Two rules borrowed straight from the red-team:

  • Re-scored fresh at compaction time. A block scored cold once, ten epochs ago, is not "continuously cold" — it's stale. Graduation requires a probe reading from this epoch, or the dwell clock resets.
  • Tool pairs are one unit. A tool_call + its tool_result (shared callId) carry one temperature and move together — the host refuses a group that orphans a pair, so the conductor never offers one.

The column — newest at top. Stable stratification: cold/deep content can't sit on warm. A unit sinks LIVE → FOLDED → COMPACTED only by cooling, and can rise back up until the moment it sediments.

3.3 · Graduation — the double-gate pipeline

LIVE
Full fidelity. Either in the protected tail or still attended-to.
→cool down
FOLDED
Coldest live units fold at a fold-epoch. Reversible. Dwell clock starts.
→graduate
COMPACTED
Sediments into a stratum. Permanent.
BOTH gates, sustained ≥ K epochs:
  • ① probe temperature below cold threshold (fresh this epoch)
  • ② agent did not recall/unfold it while folded
↩︎ Any re-warm resets the clock back to LIVE: the probe re-attends to it · the agent unfolds or recalls it · its lifetime-max temperature was ever hot (then it needs 2K epochs, not K).
Why two gates — gate ① is a 0.5B proxy and can be wrong. Gate ② is the agent's own behavior: it had the digest and the fold-tag in front of it and chose not to pull the content back. Two independent cold votes, one of them from the model that actually matters.

3.4 · Sedimentation is per-run — which dissolves the "buoyancy trap"

The compacted layer is not one descending frontier (v1's mistake — see §07). It is every maximal contiguous run of graduated-cold units, each sedimented into its own stratum. Hot or human-pinned units are buoys that bound a run; they are never buried, and the cold pockets on either side of them still sediment independently.

hot (live) warm (live) cool (folded) cold · graduated ⚓ buoy — hot / pinned / protected, halts a run ▭ a stratum forms over each cold run
Worst case dissolved — in v1 a single hot block could trap a 200-block cold region forever, and alternating hot/cold gave zero compaction. Per-run sedimentation compacts each cold pocket on its own; only a lone cold unit wedged between two buoys stays merely folded — and a single block isn't worth a stratum anyway.

3.5 · Float-up — closing attention-folder's one-directional gap

A folded unit returns to LIVE when its fresh temperature clearly out-ranks the live set's floor (a margin deadband makes this thrash-free, borrowed from tiered-relevance), or when the agent unfolds/recalls it. Compacted strata cannot float — which is exactly why graduation is slow and double-gated.

3.6 · Epochs and the honest cache-cost ladder

Hysteresis band [0.70, 0.90] of cap = min(budget, contextWindow). The v1 doc claimed the front-summary rewrite was "the cheapest cache miss." That was backwards — the front of context is the worst place to change a token. Here is the corrected story, and the design choices that follow from it:

Three events, three cache costs.
eventfrequencyLLM?cache impactdesign response
HOLDmost turnsno full hit — re-send identical commands, byte-stable prefix the default; do nothing well
FOLD-EPOCHperiodicno partial — invalid from the fold point forward maximize these to defer compaction; reversible savings are free of LLM cost
COMPACTIONrareyes expensive — a new stratum near the front invalidates forward gate hard (dwell + reversible-exhaustion). Prior strata are immutable, so we never re-pay.
the cache win that's actually true

Naive compaction rewrites its single running summary at the very front every time it compacts — a full flush, repeatedly. Thermocline's strata are append-only and immutable: a stratum is written once and never touched again, so once-compacted content is never re-invalidated and never re-summarized. We still pay one forward-invalidation when a new stratum forms — but that is rarer, and it is the deliberate price of real budget recovery, not a free lunch we pretended to have.

04Architecture

The attention signal needs a GPU process, so Thermocline is an external WebSocket conductor (the webview can't host a server or spawn Python). The decisive fact that makes the synthesis clean: external conductors can summarize with the agent's own model.

verified in protocol.ts (CONDUCTOR_PROTOCOL_VERSION 3)

cap/request { capability:"complete" } is proxied by RemoteRunner.serveCapability to the same host.complete() an in-process conductor uses — running on the user's live session model, off the model-call path. So Thermocline gets naive-compaction-grade summaries from the agent's own model, over the wire — no side LLM, no extra API keys, better quality than tiered-relevance's bring-your-own-Ollama approach.

Two out-of-band compute paths, neither on the agent's model-call path: the GPU probe (relevance) and host.complete (summaries). The wire loop itself only ever ships fold + group commands.

05Governance — locked where it must be, open where it counts

The red-team proved a fully-collaborative posture can't keep the compacted layer wire-valid (a hand-pin inside a stratum fragments the group; hand-folds corrupt the dwell clocks). So Thermocline is exclusive — but it locks the minimum, and deliberately leaves the one control that makes it better than its parent open.

controlstatewhy
human-steeringLOCKED The strata need a single owner to stay contiguous and to keep dwell bookkeeping honest. Same reason compaction-naive locks it. Consent gate on attach shows this table.
agent-unfoldOPEN — on purpose The agent's unfold is elevated to a first-class compaction veto. It is gate ②. compaction-naive locks this (the agent is powerless); Thermocline keeps the agent in the loop.
tail-sizeOPEN The human's protected-tail dial is respected. Thermocline never needs to fold into the working tail.
recall · budget · observation · detachNEVER LOCKABLE ADR 0011 floor. recall always returns a compacted block's original text as a tool result — even after sedimentation — and detach freezes the strata in place, individually reversible.
the honest concession

I wanted Thermocline fully collaborative — "a human pin means protect from compaction." The red-team showed that breaks the group contiguity invariant and the dwell clocks. Locking human-steering is a real loss of the founding "human overrides always win" posture; the compensations are that the human keeps budget + detach (a freeze, not a reset), the agent keeps a true veto, and recall can always exhume an original.

07Design evolution — what a red-team pass changed

The first cut was an attention-folder with a compaction bolted on, and it had three load-bearing bugs. Showing the deltas because the corrections are the design.

v1 · broke
One running summary, rewritten at the front each compaction. Claimed "front = cheapest cache miss."
→
v2 · fixed
Immutable append-only strata. Each cold run summarized once from originals, never rewritten. Kills recursive amnesia and the front-flush — one change, two parent-flaws gone.
v1 · broke
Single descending frontier with "buoyant floors." A lone hot block traps a 200-block cold region forever; alternating hot/cold → zero compaction; tool-pairs straddle the frontier → invalid groups.
→
v2 · fixed
Per-run sedimentation at whole-message / tool-pair granularity. Every cold pocket sediments on its own; buoys only bound runs, never trap them. Groups are always valid.
v1 · broke
Fully collaborative. A hand-pin fragments a stratum's group; hand-folds and pin/unpin cycles corrupt the dwell clocks.
→
v2 · fixed
Lock human-steering (minimum needed) — but keep agent-unfold open as the compaction veto. Beats naive compaction, stays wire-valid.
v1 · broke
One stale attention score, K quiet epochs later, drives an irreversible delete. Proxy-of-a-proxy gating permanent loss.
→
v2 · fixed
Double gate (fresh probe score and agent didn't recall) + ever-warm units need 2K + no probe ⇒ compaction disabled. Never delete on the proxy alone.
v1 · broke
Disconnect silently un-forgets the whole session — the agent suddenly re-sees content it had "forgotten."
→
v2 · fixed
Persist strata (ids + summary text + clocks) to the session file; reconnect restores them with no new LLM call.

08Honest residual risks

None of these are fixed by the design — they are the price of admission, stated plainly so the design call is informed.

serious
Attention-biased amnesia
The probe has documented recency + cross-window bias (ADR 0010). So the content we lose isn't random — it's skewed toward old, far-from-tail blocks the probe underestimates. The double-gate, ever-warm 2K rule, and recall are the net; they reduce, not eliminate, this. Irreversibility is the cost we're buying budget with.
moderate
Summary quality ≈ naive compaction
"Preserve detail proportional to temperature" is an instruction an LLM follows unreliably. Kept as a minor heuristic only; the verbatim-user-messages rule is the real guarantee, and the structural win is the immutable strata (no recursion), not magic prompt-weighting.
moderate
First-fill latency
Probe load + scoring is ~8–18 s. If context fills 0→90% faster than that, the first fold-epoch uses the FOLD_RANK fallback and compaction stays disabled until real scores exist — correct, but it means no deep recovery in the opening burst of a huge pre-existing context.
moderate
Lone-cold-unit gaps
A single cold unit wedged between two buoys never reaches a stratum (a 1-unit run isn't worth one). It stays folded — its cost is mostly recovered already, so this is an accepted ceiling, not a bug.
moderate
Persistence I/O the other conductors avoid
Strata must survive reconnect, so Thermocline writes session-keyed state to disk. That's new surface (staleness, cleanup, a session-id key) the in-process conductors never needed.
minor
Two LLM costs, not one
Probe (GPU) + per-stratum host.complete. Both are off-path and gated, but it's more moving machinery than either parent. The conductor/status readout must make in-flight state visible.

09Spec appendix

conduct() — the per-update flow (pure policy, synchronous reply)

// every context/update, against the conductor's own confirmed sets (host clears its folds each pass)
fun onUpdate(view):
  prune(foldedIds, compactedIds, dwell)              // drop ids gone / now held / protected / re-warmed
  units   = buildUnits(view.blocks)                  // tool-pairs & assistant msgs = atomic units
  visible = view.liveTokens
          − Σ(foldedIds:   tokens − foldedTokens)    // reversible savings
          − Σ(compactedIds: tokens − strataShare)    // realized savings, no double-count
  cap = min(view.budget, view.contextWindow); hi = .90·cap; lo = .70·cap

  if visible ≤ hi:                                  // HOLD — byte-stable, cache-warm
     maybePrewarmProbe(visible ≥ .80·cap)
     return emit(foldedIds, strata)               // re-send identical commands

  // ── over high-water: a FOLD-EPOCH (cheap, no LLM) ──
  scores = freshScores(units)                        // from probe; FOLD_RANK fallback if cold/absent
  for u in coldestFirst(liveCandidates, scores):
     if visible ≤ lo: break
     foldedIds.add(u); dwell[u] ??= 0; visible += foldedTokens(u) − tokens(u)
  floatUpReWarmed(foldedIds, scores)                 // thrash-free margin guard

  // ── graduate: who has earned a stratum? (double gate, ≥K epochs) ──
  for u in foldedIds: dwell[u] += (gateA(u,scores) && gateB(u)) ? 1 : reset(u)
  runs = maximalColdRuns(units, graduated = dwell ≥ K·(everWarm(u)?2:1))   // snapped to whole msgs, bounded by buoys

  if runs.empty or !host.can("complete"): return emit(foldedIds, strata)   // no probe/model ⇒ reversible-only

  // ── a COMPACTION-EPOCH (rare, one LLM call, off-path) ──
  if !inflight:
     run = oldest(runs); inflight = run
     host.complete({ system: COMPACT_SYS, prompt: build(run.originals) })   // user msgs verbatim
       .then(text => { strata.push(freeze(run, text)); persist(); reconductor() })
  setStatus("compacting " + inflight.size + " units…")
  return emit(foldedIds, strata)                   // HOLD last state while in flight

On resolve, any unit that received an agentUnfold/recall during the in-flight window is dropped from the run before the stratum is frozen — the agent's veto wins the race.

Instance state

  • foldedIds / compactedIds — explicit, separate (no view-flag inference)
  • dwell: Map<unitId, epochs> — the graduation clock
  • everWarm: Set<unitId> — lifetime-max temp was hot ⇒ needs 2K
  • strata: Stratum[] — { firstId, lastId, summary, unitIds }, immutable once pushed
  • scores, confirmedApplied/pending/rev — from attention-folder
  • vetoedDuringFlight: Set — agent unfolds/recalls mid-compaction

Config knobs

  • highWater 0.90 · lowWater 0.70 · warmWater 0.80
  • coldThreshold — temperature percentile for "cold"
  • K 3 — dwell epochs to graduate (2K if ever-warm)
  • minRunUnits 3 — don't sediment shorter runs
  • floatMargin — thrash-free deadband
  • maxSummaryTokens 8000 — per-stratum soft cap

Wire commands emitted (full desired state, every pass)

  • fold(ids) — the probation layer. Host stamps {#code FOLDED} tags ⇒ agent can unfold/recall. Reversible.
  • group(ids:[first,last], digest: strataText) — one per stratum, re-derived from the live view each pass (robust to resync). Irreversible. User messages verbatim inside.
  • Everything else: no command ⇒ live. null is never returned except as the in-flight hold, which re-sends the last batch.
